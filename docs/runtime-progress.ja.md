# 実行プロファイルと進捗表示

長時間の翻訳実行で挙動を判断するときに重要な点をまとめます。

## リリースビルド

`scripts/` 配下のテンプレートスクリプトは、通常の長時間変換をリリースビルドで実行します。

```powershell
cargo run --release -- ...
```

本番寄りの変換ではスクリプトを使ってください。ルート README の `cargo run -- ...` 例は、開発時のコマンド確認や短い動作確認向けの記述として残しています。

## ETA の計測

ETA は、前付けページや短すぎる初期サンプルで大きくぶれないよう、本文寄りの処理だけを使って遅れて表示します。

- spine 1〜3ページ目は、ETA の計測時間と文字数から除外します。
- spine 4ページ目以降で provider 翻訳が始まった時刻を ETA 計測開始とします。
- 4ページ目以降の provider 計測時間が5分に達するまでは `ETA pending` のままです。
- プログレスバーの位置は、spine 1〜3ページ目やキャッシュ済みブロックも含めて従来通り進みます。
- 選択範囲が spine 1〜3ページ目だけの場合、ETA は計測せず完了まで pending のままです。

ETA 表示開始後は、現在の実行または再開時点から測った値だけを使います。spine 4ページ目以降で完了した未キャッシュ原文文字数を provider 経過時間で割り、spine 4ページ目以降に残る未キャッシュ原文文字数へ単純に投影します。

## インラインマーカー検証

epubicus は翻訳前に XHTML のインラインタグを一時マーカーに置き換えます。

- `⟦E1⟧` と `⟦/E1⟧` はペアのインラインタグです。
- `⟦S1⟧` は自己終了インラインタグです。

provider の応答がマーカーを欠落、変更、追加した場合は、キャッシュに保存する前に拒否します。`⟦/S1⟧` のような不正な自己終了タグの閉じマーカーや、`⟦DAX⟧` のような装飾目的のマーカーも拒否対象です。古いキャッシュや batch import 由来の不正応答が EPUB 組み立てまで到達した場合も、未解決マーカーが残っていればインライン復元を失敗扱いにし、マーカーを EPUB に漏らす代わりに元の XHTML ブロックを保持して recovery 対象にします。
